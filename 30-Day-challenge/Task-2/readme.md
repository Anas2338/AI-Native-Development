# Theory

## 1. Nine Pillars Understanding

### Q1: Why is using AI Development Agents (like Gemini CLI) for repetitive setup tasks better for your growth as a system architect?

- AI Development Agents jaise Gemini CLI ko repetitive setup tasks ke liye use karna system architect ke growth ke liye is liye behtar hai kyunke yeh aapka waqt bachata hai aur aapko boring cheezon mein phansay rehne se nikal kar zyada creative aur strategic kaam par focus karne deta hai. Jab AI basic configuration aur setup handle kar leta hai, to aap architecture design, scalability, security aur problem-solving jaisi high-level skills improve kar sakte hain. Is tarah aap tez seekhte hain, zyada productive hote hain aur complex systems ko samajhne aur design karne ka mauqa milta hai, jo long-term growth ke liye bohot important hota hai.

### Q2: Explain how the Nine Pillars of AIDD help a developer grow into an M-Shaped Developer.

- Nine Pillars of AIDD aik developer ko sirf code karne wale insaan se badal kar ek “M-shaped” developer bana dete hain — matlab wo na sirf technical expert hota hai, balkay system architect, design thinker aur agent-orchestrator bhi banta hai. In pillars mein specification-driven development, AI-augmented workflows, agent orchestration, test aur quality gate, version control, human verification, iterative refinement, embedded documentation, aur production deployment shamil hain. Jab developer in sab par amal karta hai, to wo AI ko routine kaam de sakta hai aur khud creative aur strategic soch mein invest kar sakta hai, jis se uski growth aur responsibility dono badhti hain.

## 2. Vibe Coding vs Specification-Driven Development

### Q3: Why does Vibe Coding usually create problems after one week?

- Vibe coding aam tor par ek haftay baad is liye maslay paida karta hai kyunke shuru mein ideas fresh hotay hain aur coding fast chalti hai, lekin bina proper planning, documentation aur structure ke code jaldi messy ho jata hai. Phir bugs milte hain, functions samajh nahi aatay, aur agay ka kaam slow ho jata hai. Is tarah developer ko har cheez dobara dekhni padti hai, jis se frustration aur delays hotay hain.

### Q4: How would Specification-Driven Development prevent those problems?

- Specification-Driven Development in maslon ko is liye rokh deta hai kyunke is mein coding shuru karne se pehle clear plan, requirements aur steps likh liye jate hain. Jab sab kuch pehle se define ho, to code random ya vibe par nahi banta, balkay structured aur samajh aa jane wala hota hai. Is tarah bugs kam hotay hain, confusion nahi hoti, aur project ek haftay baad bhi smooth chal raha hota hai.

## 3. Architecture Thinking

### Q5: How does architecture-first thinking change the role of a developer in AIDD?

- Architecture-first thinking AIDD mein developer ka role badal deta hai kyunke is approach mein developer sirf coder nahi rehta, balkay ek system architect ban jata hai jo pehle pura design, flow aur structure sochta hai. Is mein developer AI agents ko guide karta hai, clear instructions aur specs banata hai, aur AI implementation handle karta hai. Is tarah developer ka kaam zyada strategic, organized aur long-term soch wala ho jata hai, jisse behtar systems bante hain aur developer ki growth bhi tez hoti hai.

### Q6: Explain why developers must think in layers and systems instead of raw code.

- Developers ko raw code ki bajaye layers aur systems mein sochna chahiye kyunke is tarah wo poore software ka big picture samajh sakte hain, na ke sirf choti choti lines of code. Jab aap layers mein sochte hain — jaise UI, logic, data, integration — to aapka design clean, scalable aur maintain karne mein easy hota hai. Systems thinking se aap samajh lete hain ke har hissa doosray hisse se kaise connect hota hai, jis se bugs kam hote hain, planning strong hoti hai, aur future updates smooth ho jate hain. Raw code soch sirf short-term kaam chalati hai, lekin layered aur system-level soch long-term success deti hai.


# Practical Task


### Prompt: Generate a one-paragraph specification for an email validation function that checks for an '@' symbol, verifies a valid domain like .com or .org, and returns clear, user-friendly error messages for invalid inputs.

- Ans: The email validation function will rigorously check user-provided email addresses to ensure they adhere to standard formatting requirements. Specifically, it will verify the presence of a single '@' symbol separating the local part from the domain, and confirm that the domain includes a valid top-level domain (TLD) such as'.com', '.org', or similar common extensions. For any input that fails these validation checks, the function will return clear, concise, and user-friendly error messages, guiding the user to correct their input effectively.
